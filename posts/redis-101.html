Redis 101: From a Beginner's POV

Jan 15, 2025 · 8 min read ·
Redis Databases Caching

Redis is an in-memory data structure store used as a database, cache, and message broker. It's known for being blazingly fast and incredibly versatile.

Why Redis Exists

Traditional databases are disk-based, which means they're optimized for durability at the cost of speed. Redis flips this model by keeping everything in memory, making reads and writes orders of magnitude faster.

Basic Commands

Here's a simple example of setting and getting a value: 

SET user:1 "Abdulmalik"
GET user:1
# Returns: "Abdulmalik"

Data Structures

Redis supports multiple data structures beyond simple key-value pairs:

  • Strings - Basic key-value storage
  • Lists - Ordered collections of strings
  • Sets - Unordered collections of unique strings
  • Hashes - Maps of field-value pairs
  • Sorted Sets - Sets ordered by score

Use Cases

Redis shines in scenarios where speed is critical: session storage, real-time analytics, caching, pub/sub messaging, and leaderboards.

The trade-off is that everything lives in RAM, so you need to be mindful of memory usage and have a persistence strategy if durability matters.
